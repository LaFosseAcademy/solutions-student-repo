## Brief

## Social Media Feed Application

*A small startup is developing a social media feed application that allows users to create profiles, follow friends, post updates, and like/share posts. Given their recent strategic partnerships and existing expertise, certain technology choices become clear.*

Actors:
- User: Interacts with the application to create profiles, posts, and connect with friends.
- Admin: Manages content, users, and application settings.
- Server: Hosts the application and processes requests.

Components:
- User Interface: The front end where users create posts, view feeds, and perform interactions.
- Post Database: Stores user posts, likes, and comments.
- Backend API: Manages user requests and serves data to the UI.
- Notification Service: Sends alerts for likes, follows, and comments.

Events:
- Create Post: A user creates a new post on their profile.
- Like Post: A user likes a friend's post, triggering notifications.
- Comment on Post: A user comments on a post, affecting the feed display.
- Follow User: A user follows another user to see their posts in the feed.

Additional Context:
- The startup has a team with significant experience in Java, making it more efficient to utilize Java-based frameworks.
- The startup has recently signed a partnership with Microsoft, opening access to Azure cloud resources and services, influencing their hosting choice.
